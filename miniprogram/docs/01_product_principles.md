# 01 — 产品原则

## 一句话目标

打造高中生最信赖的数学二级结论搜索工具——输入关键词，立刻拿到精确结论，不用翻页、不用等待、不用思考去哪儿找。

## 核心原则

### P1：不跳页（One Page, One Result）

- 搜索结果、详情、收藏、个人中心，每个功能一个页面完成。
- 不在页面间反复横跳——搜索结果卡片点击直接进详情，返回即回搜索位。
- 详情页内完成收藏开关，不去收藏页绕一圈。

**判断标准：** 用户完成一个目标（搜到结论→看懂→收藏），最多 2 次页面跳转。

### P2：不等待（Instant Feedback）

- 输入即响应——建议列表跟随键盘实时出现，延迟 < 100ms。
- 本地索引优先，网络是加速器而非瓶颈。远程不可用时本地照样跑。
- 详情页懒加载：先渲染首屏公式，PDF 等重资源后台缓存。

**判断标准：** 从输入到看到第一个结果，感知延迟 < 200ms。

### P3：不思考（Zero Learning Curve）

- 搜索框就是唯一入口。没有分类菜单、没有筛选面板、没有高级搜索语法。
- 输入任意关键词（中文、拼音、英文、公式符号）都能命中。
- 补全建议告诉用户"你可以搜什么"，降低认知负担。

**判断标准：** 一个新用户首次打开，不用任何引导就能完成一次搜索。

### P4：数学优先（Math-First Rendering）

- 搜索结果卡片优先展示公式，标题辅助。
- 公式渲染（KaTeX）经过微信 rich-text 兼容适配，保证手机上公式清晰可读。
- 详情页核心公式居中加粗，一眼看到。

### P5：离线可用（Offline Resilience）

- 核心索引和数据包打包进小程序，不依赖网络。
- 远程 API 是增强（热门排序、用户数据同步），不是前提。
- 当远程不可用时，本地回退必须保持一致的用户体验（只是少了云端个性化）。

## 产品路径

```
Phase 1 (当前)    单模块验证：不等式 3 条结论，本地索引，基础搜索+详情+收藏
Phase 2 (近期)    多模块扩展：三角函数、向量，搜索建议优化，远程 API 对接
Phase 3 (中期)    用户体系完善：登录、同步收藏、搜索历史、热门排行
Phase 4 (远期)    智能搜索：OCR 识别题目→匹配结论、个性化推荐、错题关联
```

## 反目标（明确不做的事）

- 不做社区/评论/UGC——工具就是工具，不做社交。
- 不做题库/刷题——定位是"查结论"，不是"做题"。
- 不做广告/商业化——当前阶段聚焦体验。
- 不做 Web/PC 端——移动端微信小程序是唯一平台。
- 二级结论之间不做超链接跳转——用返回键回到搜索，不走"相关结论"链接链。
